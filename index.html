What Gets Sanitized

Public IPs Replaced with RFC 5737 documentation IPs (192.0.2.x, 198.51.100.x, 203.0.113.x)
Private IPs RFC 1918 addresses obfuscated with synthetic 10.0.x.x addresses. Subnet masks are preserved.
IPv6 Addresses Full and compressed IPv6 addresses replaced with documentation prefix (2001:db8::) variants
Zone Names Interface zones and security zones replaced with generic names (zone-0, zone-1, ...)
Object & Group Names Named network objects, address groups, service groups, and ACL names replaced (obj-0, obj-1, ...)
Passwords & Hashes Encrypted passwords, password hashes ($1$, $5$, $6$, ENC), and enable secrets
Pre-Shared Keys IKE/IPsec PSKs, API keys, RADIUS/TACACS shared secrets, and tunnel credentials
Usernames Local admin accounts, LDAP bind DNs, RADIUS/TACACS usernames (built-in "admin" is kept)
SNMP Communities SNMP v1/v2c community strings in all vendor formats
Certificates PEM-encoded private key blocks (RSA, EC, DSA) and XML certificate key elements
BGP AS Numbers Local AS, peer AS, and autonomous-system numbers that could identify your organization
Hostnames & FQDNs Device hostname, DNS/NTP/RADIUS/syslog server FQDNs, and domain names
Email Addresses Email addresses in LDAP configs, comments, and contact fields replaced with user-N@example.net
Descriptions & Comments Free-text description fields and remark lines that may contain identifying details

Risk Disclaimer

This tool is provided on a best-effort basis and does not guarantee complete removal of all sensitive or identifying information. Firewall configurations are complex and vary widely across vendors, software versions, and deployment styles. There may be organization-specific data -- such as custom naming conventions, internal topology hints, policy structures, interface descriptions, routing details, VPN peer relationships, or vendor-specific extensions -- that this tool does not recognize or sanitize.

You are responsible for reviewing the sanitized output before sharing it externally. Always verify the output meets your organization's security and data handling requirements. This tool runs entirely in your browser and no data is transmitted externally, but the sanitized configuration should still be treated with care.
